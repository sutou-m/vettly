# T-13 書類アップロードUI（アップロード→AI処理→確認）

## 担当エージェント
@ai-engineer @frontend-developer

## 目的
書類アップロードからAI処理・候補者プロフィール確認・保存までの
一気通貫UIを実装する。Vettlyのコア機能。

## 前提チケット
- T-09・T-10・T-11・T-12完了済みであること

## 完了条件
- [ ] `/candidates/upload` で書類をアップロードできる
- [ ] ポジション選択ができる
- [ ] AI処理中にステップ表示とSpinnerが出る
- [ ] 処理結果が確認・編集可能なフォームに表示される
- [ ] 保存後に `/candidates` にリダイレクトされる

## 処理フロー

```
① 書類選択・プレビュー + ポジション選択
  ↓ 「アップロードして解析する」
② Supabase Storageにアップロード
  ↓
③ OCR処理中（Step 1/3 テキスト抽出中...）
  ↓
④ AI要約・スキル抽出中（Step 2/3 書類を解析中...）
  ↓
⑤ AIスコアリング中（Step 3/3 マッチ度を算出中...）
  ↓
⑥ 確認フォーム（氏名・スキル・スコア・要約を編集可能）
  ↓ 「候補者として登録する」
⑦ /candidates にリダイレクト
```

## ステップ表示UI
```
[●]─────[●]─────[●]
OCR    解析    スコア
```

## 確認フォームの表示項目
- 氏名（編集可）
- スコア（表示のみ・ScoreBarで表示）
- 評価軸別内訳（breakdown）
- スキルタグ（編集可）
- 職歴要約（編集可）
- 強み・懸念点（AI分析結果・参考表示）

## 注意事項
- `'use client'` で実装（複数ステップの状態管理のため）
- 処理失敗時のエラー表示と「やり直す」ボタンを実装
